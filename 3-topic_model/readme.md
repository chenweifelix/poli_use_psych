Running a structural topic model is computationally intensive and may require substantial time, especially for large datasets like ours. I ran this on a high-performance computing cluster and specified 24 cores with 30G memory. It took me about 5 hours to run. Hence, I would not recommend running this on a local machine unless you have sufficient computational resources (and have the time to spare). I provide the script here for transparency and reproducibility, but you may choose to skip running the script if computational resources are a concern. Instead, you can directly use the model files `3-topic_model/out/stm/transform_PrevFit_K30.rds` for subsequent interpreting the results.

Also, for naming the topics, I used the large language model `llama3.2` and fed the abstracts of the representative papers from each topic to generate the topic names. We are not able to provide raw text data due to data sharing restrictions; yet, I still provide the script here for transparency (`3-topic_model/src/llm_topic_naming_based_on_abs.R`). Please directly load the llm `3-topic_model/out/llm_vec_topic_term.rds` to name the topics.
